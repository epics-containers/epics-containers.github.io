Command Line Interface for IOC Management#

The python project epics-containers-cli is installed as part of the Python section of the initial tutorial Set up a Developer Workstation. It provides a command line function ec with support for:

  • building and debugging Generic IOC Containers

  • deploying, managing and debugging IOC Instances

The CLI is just a thin wrapper around the underlying tools that do the real work:

kubectl:

the command line interface to the Kubernetes APIs

helm:

the command line interface to the Kubernetes Helm package manager

podman (or docker):

CLIs for container engines

git:

the git version control system client

ec is useful because it saves typing and provides a consistent interface when working on multiple beamlines. This is because it uses the environment setup by the beamline repo’s environment.sh script. See The Environment Configuration File.

To see the available commands, run ec --help. Much of the functionality is available through subcommands dev and ioc. To get full details of a given command type the full command plus --help. For example:

$ ec dev launch --help
                                                                                 
 Usage: ec dev launch [OPTIONS] IOC_INSTANCE                                     
                                                                                 
Launch an IOC instance using configuration from a domain repo. Or by passing a
generic IOC image ID. Can be used for local testing of IOC instances. You may
find the devcontainer a more convenient way to do this.

╭─ Arguments ───────────────────────────────────────────────────────────────────╮
│ *    ioc_instance      DIRECTORY  local IOC definition folder from domain     │
│                                   repo                                        │
│                                   [default: None]                             │
│                                   [required]                                  │
╰───────────────────────────────────────────────────────────────────────────────╯
╭─ Options ─────────────────────────────────────────────────────────────────────╮
│ --execute         TEXT                 command to execute in the container.   │
│                                        Defaults to executing the IOC          │
│                                        [default: /epics/ioc/start.sh; bash]   │
│ --target          [developer|runtime]  choose runtime or developer target     │
│                                        [default: developer]                   │
│ --image           TEXT                 override container image to use        │
│ --tag             TEXT                 override image tag to use.             │
│                                        [default: None]                        │
│ --args            TEXT                 Additional args for podman/docker,     │
│                                        'must be quoted'                       │
│ --ioc-name        TEXT                 container name override. Use to run    │
│                                        multiple instances                     │
│                                        [default: test-ioc]                    │
│ --help                                 Show this message and exit.            │
╰───────────────────────────────────────────────────────────────────────────────╯

It may be instructive to understand the underlying tools and how they are are being called. For this reason ec supports a -v option to show the underlying commands being executed. e.g.

$ ec -v ps
kubectl get namespace p38-iocs -o name
kubectl -n p38-iocs get pod -l is_ioc==True -o custom-columns=IOC_NAME:metadata.labels.app,VERSION:metadata.labels.ioc_version,STATE:status.phase,RESTARTS:status.containerStatuses[0].restartCount,STARTED:metadata.managedFields[0].time
IOC_NAME            VERSION             STATE     RESTARTS   STARTED
bl38p-ea-ioc-03     2023.10.6           Running   0          2023-10-25T14:07:44Z
bl38p-ea-panda-02   2023.10.25-b16.24   Running   0          2023-10-25T15:24:41Z
